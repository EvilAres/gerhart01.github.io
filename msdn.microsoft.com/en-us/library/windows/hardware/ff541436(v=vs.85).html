Using the Hypervisor

Hypervisor Architecture

The following figure shows the virtualization architecture that is enabled by the hypervisor. The shaded areas represent components that are supplied by a vendor.

The virtualization stack runs in the root parent partition, has direct access to hardware devices, and manages child partitions. Each child partition hosts a guest operating system. Hypercalls are calls that partitions make to the hypervisor. For more information about hypercalls, see Hypercall Interface.
